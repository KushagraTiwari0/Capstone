Update the frontend UI to match an ENVIRONMENT / ECO-FRIENDLY website theme.
Focus on green, nature-inspired visuals, responsiveness, and fixing all dashboard CSS issues.

1. Environment-Based Theme (Core Requirement)
- Apply an environment-friendly color system:
  - Primary: Green shades (#2E7D32, #43A047)
  - Secondary: Leaf green / teal (#66BB6A, #26A69A)
  - Accent: Earth brown / sky blue (used minimally)
- Use soft gradients inspired by:
  - Forests
  - Leaves
  - Nature
  - Sustainability
- Avoid harsh or neon colors

2. Global Theme System
- Define all colors using CSS variables in a global file:
  --primary-green
  --secondary-green
  --eco-accent
  --background-light
  --background-soft
- Use variables everywhere (no hard-coded colors)
- Maintain calm, clean, eco-friendly UI feel

3. Dashboard CSS Fix (CRITICAL)
- Audit and fix CSS for:
  - Admin Dashboard
  - Teacher Dashboard
  - Student Dashboard
- Resolve:
  - Layout breakage
  - Misaligned cards
  - Overlapping sections
  - Incorrect spacing
  - Overflow & width issues
- Ensure stats cards, charts, tables render correctly

4. Responsive Design (Must Be Fully Responsive)
- Mobile-first design
- Breakpoints:
  - ≤ 480px (mobile)
  - ≤ 768px (tablet)
  - ≤ 1024px (laptop)
- On mobile:
  - Stack dashboard cards vertically
  - Convert tables to scrollable or card-style lists
  - Collapse sidebar into hamburger menu

5. Layout Structure
- Use CSS Grid for dashboard layouts
- Use Flexbox for inner components
- Prevent content from hiding behind navbar or sidebar
- Ensure consistent padding & margins across pages

6. Navbar & Sidebar Stability
- Fix navbar/sidebar CSS issues:
  - No jumping or shifting
  - Sidebar collapses smoothly on small screens
  - Sticky navbar behaves correctly
- Add subtle transitions (no flashy animations)

7. Eco-Style UI Components
- Dashboard cards:
  - Rounded corners
  - Soft shadows
  - Gentle hover effect (slight lift)
- Buttons:
  - Rounded
  - Green gradient
  - Clear hover & disabled states
- Use clean fonts and readable sizes

8. Loading & Empty States
- Add eco-themed loading states (simple skeletons)
- Handle empty data gracefully:
  - No broken UI
  - Show friendly messages

9. CSS Cleanup
- Remove unused and conflicting CSS
- Avoid !important
- Scope dashboard CSS to avoid leaks
- Apply global box-sizing: border-box

10. Final Visual Validation
- Test all dashboards visually
- Ensure UI remains stable on:
  - Window resize
  - Role switching
  - Empty or loading data
- UI should feel calm, modern, sustainable, and professional
